III. HTML Structure

MobyApp is based in Bootstrap 3.2 framework using 1170px with 4+ columns. If you are unfamiliar with Bootstrap grid system then you can read it here.

The main structure of the template is divided into sections, each with it’s own identifier or ID (like #home, #about, #features etc.). It is then wrapped in a div “container” followed by div “row” then your content that can be wrapped in 12 columns. Overall you can learn more about the structure I used based on Bootstrap Here .

The HTML codes are well commented so you could easy understand the structure. Just read the hide comment before or after the lines when placing your content.

IV. CSS Files

In each template layout, you will find the style.css and responsiveness.css within the css folder (android>style1>css).

I used style.css as the main css of the template. You can easily edit it to change the style of your theme such as font style, colors, and more. If you would like to edit a specific section of the site, simply find the appropriate label (classes´ names or id´s names) in the style.css file, and then scroll down until you find the appropriate style that needs to be edited.

Sometimes, you need to adjust the responsive output to make it perfect or well fitted to any mobile device and I use responsiveness.css for this. I don’t recommend editing this file, so you can make a child theme or if you are expert then you’re free to do any tweaking.

You can also find another css files at the assets folder (assets>css>colors). These are the pre-defined colors I made ready to use for you. There are 7 css files for blue, green, lightgreen, orange, pink, purple and yellow.

To use the pre-defined colors, link the colors css to the index. Example, I want blue, so I put blue.css on the index.html file below the responsiveness.css.

Make sure to place the custom color css after the style.css and not before it.

You can find more css files at bower_components folder, however these are the css files that doesn’t need of any editing. Regular updates of the following may be needed. But, please leave that to us to make sure there’s no conflicting codes.

I declared all the css files on the head section of my index.html.

V. Javascript Files

In each template layout, you will find the custom main.js file within the js folder (android>style1>js).

This file contains the entire javascript snippets I used for the template to function properly. I place comment for each snippet for you to easily determine each uses.

You can also find another js files at the assets folder (assets>js). These are use to change the map color. There are 7 js files for blue, green, lightgreen, orange, pink, purple and yellow pre-defined color scheme.

To use the pre-defined map js, link this to the index. Example, I want blue, so I put map_blue.js on the index.html file below the main.js.

Make sure to place the map js after the main.js and not before it.

You can find more js files at bower_components folder, however these don’t need any editing. Regular updates of the following may be needed. But, please leave that to us to make sure there are no conflicting codes.

I declared all the js files after the footer section of my index.html.
